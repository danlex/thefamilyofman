# Security and content concerns

## Reporting security issues

This repository is a static research wiki — it has no user accounts, no server-side code, no database, and no production secrets. The attack surface is small.

If you find a security issue in the **software** in this repository (the validation scripts, Jekyll templates, or GitHub Actions workflows), please:

1. **Open a private security advisory** via `Security → Report a vulnerability` on the repository page, or
2. Open a regular issue if the concern is minor and not exploitable.

We aim to respond within **7 days**.

## Content takedown and copyright concerns

If you believe that content hosted in this repository infringes your copyright or the copyright of someone you represent:

### Preferred — fast path

**Open a GitHub issue** labeled `dmca` with:
- The specific file path(s) or article URL(s) concerned
- A description of the work you hold rights to
- Your relationship to the rights-holder
- Your preferred resolution (attribution, takedown, license)

We will respond within **7 days** and aim to resolve cooperatively.

### Formal DMCA takedown

You can also use GitHub's formal DMCA takedown process at <https://github.com/contact/dmca>. GitHub acts as the designated agent for content hosted on GitHub and will forward requests to us.

## Our content hygiene

To minimize the chance of infringement in the first place, this project:

- Does not host the 503 photographs of the exhibition. See [`IMAGE_POLICY.md`](IMAGE_POLICY.md).
- Cites every factual claim (see [`CREDIBILITY.md`](CREDIBILITY.md)).
- Passes every contribution through a four-judge review panel before merging (see [`AGENTS.md`](AGENTS.md)).
- Scopes its own content license (CC-BY-SA 4.0) explicitly — third-party material retains its own copyright (see [`LICENSE-CONTENT`](LICENSE-CONTENT)).
- Limits third-party quotations to short passages used for scholarly commentary (fair use).

## Other concerns

For factual corrections, improvement suggestions, or editorial disagreements — which are not security issues — please use the normal contribution path:

- `✏️ Edit this page` on any article, or
- Open an issue using the [`improvement.yml`](.github/ISSUE_TEMPLATE/improvement.yml) template, or
- Start a GitHub Discussion.

See [`CONTRIBUTING.md`](CONTRIBUTING.md).
